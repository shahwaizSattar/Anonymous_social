### Post Author's View:
- **No restrictions**: Author can always react and comment on their own posts
- **No lock indicators shown**: Since they can bypass the locks
- **Full functionality**: All buttons remain enabled

## Mobile vs Desktop

### Mobile (React Native):
- Lock indicators are full-width banners
- Touch feedback disabled on locked buttons
- Toast notifications appear at top of screen

### Web:
- Same layout as mobile
- Hover states disabled on locked buttons
- Cursor changes to "not-allowed" on locked elements

## Accessibility

- Lock indicators use semantic colors (error/warning)
- Clear text descriptions ("Reactions are locked")
- Disabled state is visually obvious (greyed out)
- Screen readers will announce disabled state
- Error messages are clear and actionable
